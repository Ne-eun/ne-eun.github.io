# Promise

`Promise`는 비동기 작업의 최종 완료 또는 실패를 나타내는 객체입니다.  
콜백 지옥을 해결하고 동기 코드와 유사한 방식으로 비동기 결과를 처리할 수 있게 해줍니다.

## 1. Promise의 상태
프로미스는 다음 중 하나의 상태를 가집니다.

- **대기 (Pending)**: 이행하거나 거부되지 않은 초기 상태.
- **이행 (Fulfilled)**: 연산이 성공적으로 완료됨.
- **거부 (Rejected)**: 연산이 실패함.

이행 또는 거부 상태가 되면 프로미스가 **처리(Settled)**되었다고 합니다.

---

## 2. 주요 메서드

### 정적 메서드
- **`Promise.all(iterable)`**: 모든 프로미스가 이행하면 결과 배열을 반환하며, 하나라도 거부되면 즉시 거부합니다.
- **`Promise.allSettled(iterable)`**: 모든 프로미스의 완료 여부(이행/거부)에 상관없이 결과 객체 배열을 반환합니다.
- **`Promise.any(iterable)`**: 가장 먼저 이행된 프로미스의 값을 반환합니다.
- **`Promise.race(iterable)`**: 가장 먼저 완료(이행 또는 거부)된 프로미스의 결과를 반환합니다.
- **`Promise.resolve(value)`**: 주어진 값으로 이행하는 `Promise` 객체를 반환합니다.
- **`Promise.reject(reason)`**: 주어진 사유로 거부하는 `Promise` 객체를 반환합니다.

### 인스턴스 메서드
- **`.then()`**: 프로미스가 이행되었을 때 실행할 콜백을 등록합니다.
- **`.catch()`**: 프로미스가 거부되었을 때 실행할 에러 처리기입니다.
- **`.finally()`**: 성공/실패 여부와 관계없이 마지막에 항상 실행됩니다.

---

## 3. 핵심 특징
- **체이닝 (Chaining)**: `.then()`과 `.catch()`는 항상 새로운 프로미스를 반환하므로 연속적으로 연결하여 비동기 작업을 순차적으로 처리할 수 있습니다.
- **에러 전파**: 체인 도중 발생한 에러는 가장 가까운 `.catch()` 블록으로 전달됩니다.
