# AWS 보안그룹 (Security Group)


## 1. 보안그룹이란?
보안그룹(Security Group)은 EC2 인스턴스에 대한 가상 방화벽 역할을 하며, 인스턴스에 접근 가능한 IP 주소나 다른 보안그룹을 관리합니다.

### 특징
- 보안그룹끼리 서로 허용 대상으로 지정(합집합 형태)할 수 있습니다.
- 기본적으로 모든 인바운드 트래픽은 차단되어 있으며, 명시적으로 허용한 트래픽만 허용됩니다.

## 2. 규칙 설정

### 인바운드 규칙 (Inbound)
- 인스턴스로 들어오는(수신) 트래픽을 제어합니다.
- 허용할 포트, 프로토콜, 소스(IP 또는 보안그룹)를 지정합니다.

### 아웃바운드 규칙 (Outbound)
- 인스턴스에서 나가는(발신) 트래픽을 제어합니다.
- 기본적으로는 모든 트래픽이 허용되어 있는 경우가 많습니다.

## 3. 다른 보안그룹을 소스로 추가할 때
A라는 보안그룹의 규칙에 B라는 보안그룹을 소스로 추가하는 경우를 예로 들어보겠습니다.

![보안그룹 소스 추가 예시]({{site.url}}/assets/imgs/security_group_2.png)

- **의미**: "보안그룹 A를 할당받은 인스턴스는, **보안그룹 B를 할당받은 인스턴스로부터 오는** 트래픽을 허용한다"는 뜻입니다.
- 즉, 특정 서버들끼리의 통신을 설정할 때 유용합니다. (예: 웹 서버 보안그룹에서 DB 서버 보안그룹의 접근 허용)
